NASA Earth Exchange (NEX) Downscaled Climate Projections

This service provides the OpenNEX NEX-DCP30 data set as web-accessible data annotated with RDFa.

Demonstrations

View a demonstration that lets you browse the data via a map-based heat map.

How it Works

The demonstration uses Leaflet to display a map and then adds an overlay of a heatmap of the temperature for a particular year and month. There is a slider control with buttons to the left or right to select a particular year and month or sequence through them. At the bottom of the page, there are buttons for selecting one of four data sets produced for NEX-DCP30.

The heat map application reads the temperature data, received as tables with RDFa annotations, directly from the Web, and uses simple SVG overlays to draw onto the map. All the rendering, including the mashup with the underlying map, zooms and scrolling, is done on the client. The application uses the 1/2 degree data, but it would only take a small change to use a different scale, or allow changes between scales.

The raw data is accessible from the same Web server as the Web application. This is not a requirement---the application can be hosted anywhere, as the data services all use CORS.

Data Access

The data is stored in the native grid resolution of 1/120° but it is summarized in 1°, 1/2°, 1/4° quadrangles. Each summary contains a link to a partition (a quadrangle) of the data whose dimensions are of the same size (e.g., 1/2° summary points to 1/2° quadrangles). The partition is labeled with a quadrangle sequence number.

Each summary is accessible at:

./data/{name}/{yearMonth}/{resolution}/

where:

  • name — a data set name (e.g., 'avg-rcp85')
  • yearMonth — the year and month for the summary (e.g., '2014-10')
  • resolution — the summary size in 1/120° increments that must be one of 30, 60, or 120 for 1/4°, 1/2°, 1°, respectively

Each quadrangle partition is accessible at:

./data/{name}/{yearMonth}/{resolution}/{partition}

where partition is the sequence number of the quadrangle.

For example, to access a 1/2° summary for 2014-10 of the average model, you use:

./data/avg-rcp85/2014-10/60/

The returned summary has links to various partitions. These partitions are accessible individually:

./data/avg-rcp85/2014-10/60/60235
