Problem Description

  • Often, data formats used by professional science
  • are difficult to process
  • and inaccessible on the Web
  • while possibly justified
  • they leave out other government
  • and business agencies
  • as well as citizen scientists.
  • We need to enable
  • access to data sets
  • from within the technologies
  • of the Open Web Platform
  • so that everyone
  • has access to scientific data.

Summary of Solution

  • The OpenNEX NEX-DCP30 data has been transformed
  • into Web accessible data
  • using RDFa as a semantic web encoding
  • partitioned into quadrangles
  • so that it is sized appropriately
  • for use within Web
  • by client-side browser applications.

View a demo online!

Use of Datasets

  • The OpenNEX NEX-DCP30 climate data is produced at resolution 1/120°:

    • each partition (5 years) is about 1.3GB
    • each month is a smaller slice
    • that can be made smaller by handling non-data (e.g., fill values)
  • Lower resolution summaries (e.g. 1/2°) can provide quick visualizations requiring less space.
  • Data needs to be provided for quick access by Web-based application.

  • Use of CORS enables access:

    • Cross-orign resource sharing (CORS)
    • enables data services
    • to be accessed by any website.
  • RDFa annotations provide scientific contexts (e.g. quantities, domains, locations, etc.)

Solution Details

  • Data is extracted from the OpenNEX datasets
  • by utilizing map reduce on AWS EMR (Hadoop)
  • to reorganize, summarize, and store
  • data into AWS S3
  • exposed by web services
  • hosted on AWS EC2 instances
  • served up to various browser applications
  • encoded via RDFa
  • accessed via CORS requests
  • and render by browser applications.
