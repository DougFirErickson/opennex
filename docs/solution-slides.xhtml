OpenNEX on the Web

October 21, 2014

A project to summarize and partition the OpenNEX data sets for use within the Open Web Platform (OWP). The solution provides the data infrastructure to enable Web applications to use NEX-DCP30 data set within the browser for client-side rendering and computation.

Problem Description

  • Often, data formats used by professional science
  • are difficult to process
  • and inaccessible on the Web
  • while justified for their own purposes
  • they leave out other government
  • and business agencies
  • as well as citizen scientists.
  • We need to enable
  • access to data sets
  • from within the technologies
  • of the Open Web Platform
  • so that everyone
  • has access to scientific data.

Summary of Solution

  • The OpenNEX NEX-DCP30 data has been transformed
  • into Web accessible data
  • using RDFa as a semantic web encoding
  • partitioned into quadrangles
  • so that it is sized appropriately
  • for use within Web
  • by client-side browser applications.

Use of Datasets

  • The OpenNEX NEX-DCP30 climate data is produced at resolution 1/120°:

    • each partition (5 years) is about 1.3GB
    • each month is a smaller slice
    • that can be made smaller by handling non-data (e.g., fill values)

  • The OpenNEX NEX-DCP30 data is:

    • Processed via map reduce on AWS EMR (Hadoop)
    • to summarize, partition, and store
    • into AWS S3
  • Goal is to provide quick access to Web-based application.
  • RDFa annotations provide scientific contexts (e.g. quantities, domains, locations, etc.)
  • Demonstration Web browser applications shows the use of data within visualizations as a browser-side computation.

Solution Details

Use of CORS enables access:

  • Cross-origin resource sharing (CORS)
  • enables data services
  • to be accessed by any website.

Web services

  • hosted on AWS EC2 instances
  • serve up DATA to various browser applications
  • encoded as RDFa data.

Data extraction process runs one time only and results are re-used.
